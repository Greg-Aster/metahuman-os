# Graph JSON Persistence Migration

This document tracks the cleanup of saved cognitive graph JSON files after the
node-defaults and graph-editor conversion pass.

## Goal

- Preserve each saved graph's executable workflow shape.
- Preserve per-node `data.properties`, including user-editable prompt and
  numeric overrides.
- Remove editor/runtime-only node fields that should be regenerated by the
  graph editor or runtime instead of persisted in workflow JSON.
- Make `pnpm validate:node-defaults -- --strict-graphs` pass.

## Removed Fields

Only these node `data` fields were removed:

- `schema`
- `executionState`
- `executionOutput`

These fields are editor/runtime metadata. Node identity remains in
`data.nodeType`; editable workflow values remain in `data.properties`.
